Collection集合

为什么要有集合

  • 数组的长度是不可变的，对数组扩容缩容不方便操作，集合的长度是可边的，自动扩容缩容
  • 集合的操作比数据更加方便快捷

集合的分类：

  • 线性集合：Collection父接口集合，实现了改接口的都是线性集合
    • list子接口：常用实现类：
      • ArrayList：
      • LinkLIst
    • set子接口：常用实现类HashSet和TreeSet
  • 键值对集合：Map

List集合：有序的，可以有重复元素

Set集合：无序的，元素不可重复

List集合（有序可重复）

两个常用的实现类内存数据结构对比：

  • ArrayList底层数据结构是数组：具有数组数据结构的List集合，由下表，可以通过下标获取或者删除修改集合中的元素，下标和数组一样从0开始；

    与数组不同的是，ArrayList的长度是可变的，添加元素和删除元素不需要手动扩容缩容；

  • LinkedList底层数据结构是链表：上香链表结构没有下标，添加数据先断开链表的连接，在双向连接；

两个常用实现类用法对比：

  • 查询方面：

    • ArrayList有下标，可以根据下标直接获取元素，效率高

    • LinkedList没有下标，获取元素需要前后遍历，查询效率低

  • 修改方面：

    • ArrayList删除和添加需要移动元素，效率低
    • LinkedList删除和添加只需要断开连接然后在断开的地方删除或者添加元素，效率比ArrayList高

ArrayList与LinkedList小结

数据结构 查询（对比） 修改（对比）
ArrayList 数组 快 慢
LinkedList 链表 慢 快

List常用的方法：

定义一个List集合

List list = new ArrayList();
LinkedList list1 = new LinkedList();

ArrayList

  • 向集合中添加元素

    list.add("字符串");
list.add(123);
//list集合的有序是指集合的元素是按照存入的顺序排列的

  • 在指定位置插入元素

    list.add(index,元素);//在下标index处插入元素
list.add(2,"哈哈");

  • 获取集合的长度

    list.size();

  • 获取集合中的元素

    list.get(index);//index就是下标

  • 移除集合元素

    list.remove(index);//index就是下标

  • 更改指定位置的元素

    list.set(index,新元素);//在下标index处，将原来的元素改为新元素
list.set(2,"新字符串");

LinkedList（前面ArrayList的方法都适用）

  • 获取元素的头节点

    list1.getFirst();
list1.element();//在源码中，这个方法也是直接调用的getFirst方法

  • 获取元素的尾节点

    list1.getLast();

  • 删除头节点

    list1.poll();//只能删除头节点，里面不能加参数
list1.remove();//不加参数默认删除头节点，加上参数，则删除对应下标的元素

遍历集合

foreach循环遍历集合

//此时还没有给集合加上一个确定的类型，所以集合中的元素先用object
for(Object obj:list){
    system.out.println(obj);
}

set集合（无序不可重复）

set集合一般有两种实现类：

  • HashSet实现类：散列集合，底层是哈希表；保证元素唯一性的原理就是通过hashcode和equals方法，无序是指存入的数据不是按照顺序存入的，输出的顺序会打乱

  • TreeSet实现类：是树状结构，存放可以有顺序

HashSet常用方法

  • 创建一个hashSet集合
Set set = new HashSet();

  • 添加元素

    set.add(元素1);
set.add(元素2);

  • 判断集合是否包含某个元素(boolean类型)

    set.contains(参数);//set集合中，是否存在一个元素与该参数匹配（按照整体对比）

  • 删除元素

    set.remove(参数);//因为set集合是没有小标的，只能给定元素来删除，返回值是boolean类型，是否删除成功

  • 清空集合（无返回类型）

    set.claer();

  • 判断集合是否为空

    set.isEmpty();

遍历集合

forearch循环

for(Object obj:set){
    system.out.println(obj);
}

迭代器遍历（Iterator）

//步骤1：根据集合获取对应的迭代器
Iterator it = set.iterator();
//步骤2：判断是否右下一个元素，有的话执行循环，没有的话结束迭代
while(it.hasNext()){
    //步骤3：去除光标后面的元素
    system.out.println(it.next());
}

Map集合(无序存放，以键值对存储数据)

无序存放，以键值对（key-value）结构存储数据，key不可重复，一个键对应一个值；key相当于一个识别码，通过key找value值

Map的常用方法

  • 建立一个Map集合

    Map map = new HashMap();

  • 存入数据（put）

    map.put(键,值);//键（key），值（value）,key不可重复，一个键对应一个值

  • 获取map的长度(int类型)

    map.size();

  • 通过key获取value

    map.get(key);//获取到key对应的值（value）

  • 查找map中是否存在某个键

    map.containsKey(key);

  • 查找map中是否存在某个值

    map.containsValue(value);

  • 获取集合中的所有元素的key（返回Set集合）

    Set set = map.keySet();
//用forearch遍历key集合获取value的值
for(Object obj:set){
    system.out.println(map.get(obj));
}

Entry+Interator遍历

//第一步，先将map集合中的key，value作为一个整体放入set中；相当于将map集合转为set集合
Set set = map.entrySet();
//第二步，使用迭代器遍历
Interator it = set.interator();
while(it.hasNext()){
    //第三步，将set中的元素转为key，value格式
    //这里的entry就是键值对存储数据的数据类型
    Map.entry entry = (Entry)it.next();
    //key:entry.getKey()
    //value:entry.getvalue();
system.out.println("key:"+entry.getkey()+"value:"+entry.getValue());

泛型的作用

在编译器中检查类型的约束，用再集合上，可以用来约束集合中元素的数据类型，泛型尖括号中只能指定一种类型，不能指定多种

注意：泛型括号中要使用基本数据类型的包装类或者其他引用类型

泛型在集合中的应用

//使用尖括号来添加指定泛型，来约束集合，让这个集合只能存放特定的类型数据
//如果在使用该list，向里加其他类型的数据，编译器就会报错
List<String> list = new ArrayList<String>();
//在使用集合的时候，建议给集合添加对应的泛型，让集合的元素数据类型有明确的约束
//在Map集合中的泛型，key和value都需要指定的类型
Map<String,Double>maps = new HashMap<String,Double>();
//获取key的值的集合，也可以指定泛型
Set<String> sets = maps.keySet();
//带泛型之后，使用forearch迭代器就可以指定类型不用Object了
for(String str:sets){
    system.out.println(str);
}

使用泛型后，entry遍历集合

//将Map转成Set，将Map中的兼职对作为一个整体，存放入set集合
Set<Map.Entry<String,Double>> sets = maps.entrySet();
//遍历转换后的Set，将每个key-value都挨个取出来
//这里的Map.Entry是作为一个数据类型，里面存的Key-Value键值对
for(Map.Entry<String,Double> e:sets){
    System.out.println(e.getKey()+","+e.getValue());   
}

泛型中指定对象，让集合中存放指定类型的对象，带泛型后，最好是等号两边泛型保持一致

泛型的高级应用

  • 在创建类时，在类的后面给定一个泛型

    public class Tdemo<T>{
    private T age;
}
//T只是这个泛型的代指，一般用T来表示

  • 在创建对象的时候就可以指定T的类型了

    Tdemo<Integer> t = new Tdemo<Integer>();

总结

  • Comparator中的compare方法，需要一个单独的类，让这个类的对象作为比较的方式，对有两条具有课比较的数据进行比较
  • Comparable中的compareTo方法，不需要单独写一个比较的类，是让本身和需要比较的传进来的对象比较
  • 两个方法的返回值都是int
    • 如果返回值是负数，那么认为需要比较的两个数据，前一个小
    • 如果返回值是正数，那么认为前一个大
    • 如果返回0，那么就是相等的
