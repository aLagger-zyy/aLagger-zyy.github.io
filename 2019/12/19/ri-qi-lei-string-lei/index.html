一、日期类

1.long类型的时间（时间的毫秒数）

使用情况较少

1.1 获取当前的时间

获取到当前日期的毫秒数 ，long类型

long time = System.currentTimeMillis();

1.2 输入毫秒数

直接输出time，得到的是一长串数字，即当前时间的毫秒数

System.out.println("当前的毫秒数"+time);

1.3 将毫秒换算成年份

//当前时间2019年12月
//除以1000毫秒，除以60秒，除以60分钟,除以24小时,除以一年365天
int year = (int)(time/1000/60/60/24/365);

这里的year时间是输出49，因为计算机中时间的其实从1970年开始

2.date类型的时间

因为千年虫问题，所以很多方法过时

2.1 获取当前的时间

Date date = new Date
System.out.println("日期为"+date);
//输出日期为Tue Dec 10 21:17:03 CST 2019

2.2 获取年份月份

获取年份

这个方法会有横线，代表这个方法已经过时了，很少被用到，因为有千年虫的问题，导致结果不对

System.out.println("年份"+date.getYear());//输出119
获取月份
System.out.println("年份"+date.getMonth()+1);//输出12

3.calendar（格林威治时间）

3.1 创建格林威治时间对象

获取格林威治的时间有两种方式

//第一种：通过子类创建格林威治时间对象
Calendar cal = new GregorianCalendar();
//第二种：使用工厂方法创建对象，不使用new，这是系统给的另一个创建日历类对象的方法
//推荐使用
Calendar cal = Calendar.getInstance();

System.out.println(cal);
//输出cal:java.util.GregorianCalendar[里面是各种参数的值]

3.2 获取具体的时间

首先可以将格林威治转换为Date类型，也可以转换成long类型，然后直接输出

//转换为Date类型后，可以使用Date类型中的方法
Date date = cal.getTime();
System.out.println(date);//Tue Dec 10 21:27:15 CST 2019
//转换为Long类型，获取的是时间的毫秒数，从1970年到现在
long times = cal.getTimeInMillis();
System.out.println(times);//1576860071198

获取到具体的时间(get()方法)

注意：

  • 月份的范围是0-11（即一月为0）
  • 星期的范围是0-6且星期的顺序为（周日0，周六6）
  • get( 参数)里的参数是int型的数据，返回值也是int类型
Calendar cal = Calendar.getInstance();
//获取到年份
cal.get(Calendar.YEAR);
//获取到月份（范围是0-11）
cal.get(Calendar.MONTH);
//获取到当前时间是这个月的第几天
//两种办法
cal.get(Calendar.DAY_OF_MONTH);
cal.get(Calendar.DATE);
//获取到当前时间是这周的第几天（范围0-6）
cal.get(Calendar.DAY_OF_WEEK);
//获取当前时间所在的周在这个月的第几周
cal.get(Calendar.DAY_OF_WEEK_IN_MONTH);
//按照12小时制获取当前的小时
cal.get(Calendar.HOUR);
//按照24小时制获取当前的小时
cal.get(Calendar.HOUR_OF_DAY)
//获取到分钟
cal.get(Calendar.MINUTE);
//获取到秒数
cal.get(Calendar.SECOND);

修改时间（set，add方法）

set方法

参数列表：(需要修改的时间，修改后的数据)

//修改年份
cal.set(Calendar.YEAR,2018);
//修改月份为2月
cal.set(Calendar.MONTH,1);
//再将当前月份的日期修改为31号
cal.set(Calendar.DAY_OF_MONTH, 31);
//这个时候会输出3月3号，日历会根据每月的天数，自动跳到次月

add方法

参数列表：（需要修改的时间，需要往时间上加的数据）

cal.add(Calendar.YEAR,2);//当前的年份2019，+2，变为2021
//如果需要时间回退，那只需后面的参数变成负数就可以了

5.日期类与字符串的互相转换

日期类与字符串相互转换，主要用到SimpleDateFormat转换工具类

字符串转Date用parse

String str = "2019-12-12 20:20:20";
//y:代表年
//M:代表月份
//d:日
//E:星期
//a:上午，下午
//H:小时
//m:分
//s:秒
SimpleDateFormat format = new SimpleDateFormat("yyyy-MM-dd HH:mm:ss");
Date date = format.parse(str);

日期类型转化为字符串用format

Date date = new Date();
//同样使用SimpleDateFormat类转换，先定义转换格式
SimpleDateFormat format2 = new SimpleDateFormat("yyyy/MM/dd HH/mm");
//使用format方法将日期转换为字符串
String str2 = format2.format(date);

日期类型中的时区

//设置时区
TimeZone zone = TimeZone.getTimeZone("GTM+8");
//设置时间输出格式
SimpleDateFormat format3 = new SimpleDateFormat("HH:mm:ss");
//设置时区
format.setTimeZone(zone);

6.三种日期类型的互相转换

long转Date转Calendar

//long先转Date
long longTime = System.currentTimeMillis();
Date date = new Date(longTime);
//Date再转Calendar
Calendar cal = Calendar.getInstance();
cal.setTime(date);

Calendar转Date转long

Calendar cal = new GregorianCalendar();
//calendar转Date
Date date = cal.getTime();
//Date再转long
long longTime = date.getTime();

7. 定时器

//创建定时器
Timer timer = new Timer();
//定时器的任务调度，自动执行
timer.schedule(new TimerTask() {
    //匿名内部类，重写run方法，定时器需要执行的操作
    @Override
    public void run() {
        Date date = new Date();
        //转换输出时间
        SimpleDateFormat format = new SimpleDateFormat("HH:mm:ss");
        String str = format.format(date);
        System.out.println(str);
    }
},0,10000);//这里的参数，开始时间，每次调度任务执行的间隔
//每隔10秒输出一次时间

二、String类

1-创建字符串

  • 第一种，直接创建str并赋值

    String str = "123";

  • 第二种，使用new

    String str = new String("123");
    • 这段代码创建了一个或者两个对象
    • 方法区中字符串常量池中查询是否有”123”这个字符串，如果有之间创建str对象然后指向"123",如果没有"123"就会先创建一个"123"然后再创建str，再将str指向"123"

  • 第三种，使用字符数组创建字符串

    char[] chs = {'1','2','3'};
String str = new String(chs);

2-字符串拼接的两种方式

  • 直接使用 + 拼接

  • 使用concat方法

    String str1 = "abc";
String str2 = "def";
str1.concat(str2);

3-String常用API

  • 获取指定位置上的字符，返回char型

    str.charAt(2);//返回char类型，字符串下标从0开始
str.cahrAt(str.length()-1);//获取最后一个字符

  • 获取子字符串第一次出现的位置，返回int型

    str.indexOf("字符串");//返回int类型,参数中放入一个字符串，获取该字符串第一次在str中出现的位置（这个位置是字符串之首在str中的位置）
str.indexof("字符串",8);//从str的第8位开始(包括第8位)，向后查找字符串第一次出现在str中的位置
//注意，如果没有，则返回-1；

  • 判断字符串首尾的内容，返回boolean型

    str.startsWith("123");//起始位置
str.endsWith("abc");//初始位置

  • 将字符串转换成字符数组

    char[] ch = str.toCharArray();

  • 判断字符串是否包含指定

    str.contains("字符串");//返回boolean
str.indexOf("字符串");//查找不到返回-1

  • 截取字符串，*返回String类型 * 

    //substring方法中只加一个参数那么就是从下标开始截取到末尾
//如果有两个参数，那么截取的是指定参数区间内的子字符串[参数1，参数2)
str.subString(5,10);

  • 字符串的分割，返回String数组

    String str = "192.168.0.4";
//参数使用正则表达式
String[] str1 = str.split("\\.");//按小数点分割
//str1中存放{"192","168","0","4"}

  • 大小写转换，返回的是String类型

    str.toLowerCase();//将英文全部转为小写
str.toUpperCase();//将英文全部转为大写

  • 去除头尾空格

    str.trim();//去除头尾的空格

  • 替换字符串中的元素

    str.replace(参数1,参数2);//将字符串str中的参数1全部替换成参数2
str.replace(参数1,参数2);//参数1可以是正则表达式，也可以是字符串，将参数1全部替换成参数2
//当然，如果要移除str中的所有的参数1，也可以把参数2写成"";
str.replace(参数1,"");

4-判断字符串相等

String strA = new String("tom");
String strB = new String("tom");
strA == strB//false,比较的是地址
strA.equals(strB)//true,比较的是字符串的值
String strA = "tom";
String strB = "tom";
strA == strB//true,地址相同
strA.equals(strB)//true,内容相同
//字符串存在字符串常量池中,因为没有new，所以strA跟strB的地址都指向
String strA = "hello";
String strB = new String("World");
String strAB = strA+strB;
String strC = "hello"+"World";
String strD = "helloWorld";
strAB == strD//false，使用new创建字符串，会在堆空间中开辟空间存储地址并指向"World"与7直接指向常量池不同
strC == strD//true，strC在字符串常量池中，拼接成helloWorld,系统会自动地将地址指向helloWorld，所以地址是一致的
String a = "hello";
String b = "World";
String ab = a+b;
String c = "helloWorld";
c == ab//false,只有使用引号包含文本的方式创建的String对象之间使用“+”连接产生的新对象才会被加入字符串池中,而ab并没有将两个字符串在常量池中创建helloWorld，所以ab的地址并没有被引到helloWorld，所以地址值不相等

5-String,StringBuffer,StringBuilder

String字符串是不可变的

String str = "甜甜";
str += "喝奶茶";
//String字符串的不可变是指String常量池中存储的字符串是不可变的
//String str = "甜甜";str的地址会指向常量池中的"甜甜"
//str += "喝奶茶";str的地址会指向常量池中的"甜甜喝奶茶"，而不是将常量池中的"甜甜"变更为"甜甜喝奶茶"，是换了一个地址；
//"甜甜"一旦在常量池中生产出来，那么这个数据就是不可变化的

注意：字符串缓冲区（字符串常量池）：当定义字符串时，首先会去字符串常量池查找是否有 该字符串的对象，如果有会直接引用该对象，没有就会创建该字符串对象

StringBuffer与StringBuilder

StringBuffer是一个线程安全的可变字符串序列，它是一个类似于String的字符缓冲区，其实体容量会随着字符串改变而自动改变

1.StringBuffer，StringBuilder中的特殊API：
StringBuffer sbf = new StringBuffer("门前大桥下");
//字符串的追加
sbf.append("游过一群鸭");
//向指定位置插入字符串，两个参数  位置和需要插入的字符串
sbf.insert(5,",")//在下标为5的位置上插入一个","，之后的字符串依次后移
//修改指定索引出的字符，参数为 位置，需要修改的字符
sbf.setCharAt(sbf.length()-1,'鱼');//最后一位替换为鱼
2.String，StringBuffer与StringBuilder中的相互转换
//String转StringBuffer/StringBuilder
StringBuffer sbf = new StringBuffer(str);
//StringBuffer/StringBuilder转为String
sbf.toString();
//StringBuffer/StringBuilder互相转换需要先转为String再转为需要的类型
StringBuffer sbf = new StringBuffer("字符串");
StringBUlider sbd = new StringBulider(sbf.toString());
3.三者的区别
String StringBuffer StringBulider
对象类型 字符串常量，不可变 字符串变量，可变 字符串变量，可变
线程安全性 不安全 安全 不安全
执行效率 效率最低 效率比StringBulider低 效率较高

小提示：

  • 在实际的项目开发中，重写toString方法时，在连接字符串的时候，使用StringBulider连接

  • String是基本数据类型，StringBulider跟StringBuffer是引用数据类型

总结：

  • 操作少，数据少用String
  • 单线程，操作数据多，用StringBulider，单线程不存在线程安全
  • 多线程，操作数据多，用StringBuffer，防止多线程的安全性问题
